              Description

              We had several useful extensions for PicoContainer on the list, that I would like to collect in a picocontainer-gems "component". Such an extension should enhance the core pico functionality, that should not bloat the core itself. I would like to use this project also as new home for some other code already in the CVS. As outline the extensions should not use nanocontainer nor use GUI classes.

              Proposed contents of picocontainer-gems:

              • LCListener from Graham Lea from the list
              • CollectionParameters and Contraints from Nick Sieger (PICO-206)
              • StaticFactoryCA from Pico
              • ThreadLocalCA from Nano
              • merge proxytoys into gems
              • merge consolidated pool into gems
